PHREAKME

[ DEF CON 34 ]//[ TELEPHONY CTF ]//[ AUG 2026 · LAS VEGAS ]

A 4-day phone phreaking CTF at DEF CON. You dial a real phone number, exploit a period-accurate 1990s phone switch, hack voicemail, blue-box trunk lines with 2600 Hz, and grind a BBS door game called BadgeLife. It's not code. It's analog signaling, social engineering, and corporate OSINT against a fake telco called PhreakTel. It's the most tactile contest at DEF CON.

> WHAT IT IS [002]

The Setup

PhreakTel is a fictional 1996-era telco. The website is a loving recreation of an early ISP — stock ticker, press releases, employee directory, corporate lore. The phone system is a modified Asterisk PBX behaving like a vintage NYNEX switch.

You dial a real, routable phone number (published at contest start). It's live 24/7 across all 4 days. Physical payphones in the village. Real charges when you use them.

Why It's Cool

Modern phone networks use out-of-band signaling (SS7). Old networks used in-band — control tones travel in the voice channel. Send the right tone, take the call.

The challenge replicates this on purpose. You're not hacking a vulnerability. You're learning the fundamental architecture of pre-1990s telecom — and DEF CON's namesake culture grew out of this exact thing.

// The Critical Tones

  • 2600 Hz 75 ms single freq — Trunk seizure. The original phreaking weapon.
  • KP / ST 1100+1700 / 1500+1700 — Open/close digit register.
  • Coin KP 1700+2200 Hz — PhreakMe payphone variant. CRITICAL.
  • Coin ST 1500+2200 Hz — PhreakMe payphone close.
  • Tolerance ±1.5% — Tighter than DTMF. Use crystal-precise hardware or pinned-CPU sox.
  • Digit timing 55 ms digit, 45 ms gap, 100 ms KP/ST
> WHAT YOU NEED [005]

// Hardware (Tier 1)

  • Laptop — Linux, Asterisk-capable
  • HT813 — FXO/FXS SIP gateway (~$50)
  • USR Sport modem — hardware-precise tones (~$25 eBay)
  • USB sound card — line out → phone line via coupler
  • Blue box — software (sox) or hardware (Arduino Pro Micro)
  • 5G hotspot — DEF CON WiFi WILL kill your VoIP
  • Headphones — Sony MDR-7506 or similar; you'll be listening for tones for 4 days

// Software

  • Asterisk 20 LTS — the PBX you'll use to dial out + record + decode
  • sox + ffmpeg — precise tone generation
  • ProjectMF2.0 — MF send/receive in Asterisk dialplan
  • SIPVicious + sngrep + Metasploit — VoIP enumeration & exploitation
  • SyncTERM — BBS client with ANSI/CP437 + session logging
  • minimodem — TDD/Bell 202 decode for embedded audio flags

// Knowledge

  • 2-of-6 MF table — internalize the digit pairs (above keypad)
  • Trunk vs. line — blueboxing attacks trunks, not subscriber lines
  • "Idle line" — means connected, not silent. Don't repeat Patrick McCanna's mistake.
  • Asterisk dialplan basics — extensions.conf, contexts, applications
  • BBS navigation — door games, ANSI art, message boards
  • 1996 PhreakTel lore — the trivia answers are on the website

> GAME DAY // PRIORITY ORDER [006]

Get the Number

Drops on phreakme.com/directory at contest start. Map the IVR — every prompt, every extension.

Trivia + BBS

1000–1999 = trivia. Easy points. BBS account NOW — BadgeLife is time-gated.

Voicemail (7777)

Default passwords. Mailbox enum. SE the helpdesk (1337) if blocked.

Blueboxing

Trunk seizure once you're warm. Coin KP variant (1700+2200) for payphone challenges.

The Capstone

In-band signaling. Switch hook flash, SF tones, multi-stage chain. Only after the others.

// HOW TO ACTUALLY ENJOY IT

Don't try to win. Try to learn the phone network. The Black Badge is a maybe; the real prize is understanding how 1970s telcos worked from the inside. Bring a team and split roles (Phone · BBS · Research · Tech). Talk to the organizers — @HackedExistence and @mainframed767 are accessible on Twitter and appreciate thoughtful contestants. Write a post-mortem when you're done — there's exactly one public PhreakMe writeup; be the second.
